STEP 2 · GET A FREE GOOGLE GEMINI API KEY (5 MINUTES)
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

The automated CA pipeline uses Google Gemini AI to:
  • Write a UPSC-specific 2-3 sentence summary of each news article
  • Tag the GS paper (GS1/GS2/GS3/GS4/Essay)
  • Extract subject tags and syllabus links
  • Generate one MCQ quiz question from each article

WITHOUT the key: raw RSS article titles and descriptions are saved.
  You will need to manually write summaries and tags before publishing.
WITH the key: articles arrive pre-processed. You just review and publish.

How to get the key (free, no credit card):
  1. Go to  https://aistudio.google.com/app/apikey
  2. Sign in with your Google account
  3. Click "Create API key" → "Create API key in new project"
  4. Copy the key (starts with "AIza...")

Free tier limits (as of 2026):
  • gemini-1.5-flash: 1,500 requests/day, 1 million tokens/minute — MORE than enough
  • We fetch ~20-30 articles/day; each uses ~1 request

Where to add it:
  Edit  server/.env  (copy from server/.env.example if it doesn't exist)
  Add this line:
    GEMINI_API_KEY=AIzaSyXXXXXXXXXXXXXXXXXXXXXXXXXXXXXXXXX

  Then rebuild:  docker compose up --build

STEP 3 · UNDERSTAND HOW AUTOMATION WORKS (READ ONCE)
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

Every morning at 07:30 IST, the server automatically:

  1. Fetches articles from 3 RSS feeds:
       - The Hindu (national news)
       - PIB (government press releases)
       - Indian Express (national news)

  2. Filters out duplicates (same title already in DB today)

  3. For each NEW article:
       IF Gemini key is set:
         → Sends title + description to Gemini
         → Gemini returns: UPSC summary, GS tag, subject tags,
           syllabus links, and one MCQ quiz question
         → Saves CurrentAffair row (is_published = FALSE)
         → Saves QuizQuestion row (is_approved = FALSE)
       IF no Gemini key:
         → Saves raw RSS data (is_published = FALSE)

  4. YOU log into Swagger or admin panel, review items, set is_published = true

  5. The CurrentAffairs page and Dashboard instantly show published items.

Scheduler also runs these automatically:
  • 06:00 IST daily    — rotate daily answer-writing question
  • 00:05 IST daily    — reset daily study leaderboard
  • Monday 00:10 IST   — reset weekly study leaderboard
  • Every hour         — expire silent library sessions open > 8 hours

STEP 4 · YOUR DAILY EDITORIAL WORKFLOW (5-10 MINUTES/DAY)
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

After 07:30 IST every day:

  A. Open Swagger UI:  http://localhost:8000/docs
     (In production: https://your-api-domain.com/docs)

  B. Authorize: click the padlock icon → enter admin credentials
     Email:    admin@demo.drishti.dev
     Password: Demo123!
     (Change these in production — see Step 7)

  C. Review today's ingested items:
     GET /api/v1/affairs/?language=en&limit=50
     Look for items with is_published = false

  D. For each item you want to publish:
     PUT /api/v1/affairs/{affair_id}
     Body: { "is_published": true }

     Optionally edit:
     { "is_published": true, "detailed_notes": "Your analysis here...",
       "syllabus_links": "GS3: Indian Economy" }

  E. For quiz questions generated by Gemini:
     GET /api/v1/quiz-questions/?language=en&limit=50
     (is_approved=false items need your review)

     PUT /api/v1/quiz-questions/{question_id}
     Body: { "is_approved": true }

  F. For daily answer-writing question:
     If the queue is running low (< 7 items), add more:
     POST /api/v1/daily/questions/
     Body: { "title": "...", "content": "...", "subject": "GS2", "word_limit": 250, "marks": 15, "is_active": false }

Total time: ~5 minutes/day with Gemini key. ~15 minutes without.
